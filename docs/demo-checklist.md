# 대회 시연 전 점검

## 전날

- Render 최신 배포가 성공했는지 확인합니다.
- `npm run verify:live`로 서버, API 키 상태, HACCP DB, 국내 샘플 조회를 검사합니다.
- 실물 검증을 마친 국내 제품 2개와 서로 다른 포장 재질 제품을 준비합니다.
- 카메라 권한을 허용하고 후면 카메라 전환 여부를 확인합니다.

## 시연 10분 전

- 웹 앱을 한 번 열어 Render 서버를 준비 상태로 만듭니다.
- 급식 조회에 사용할 학교와 급식이 있는 날짜를 확인합니다.
- 국내 샘플 바코드 조회와 푸드 캘린더 저장을 한 번 실행한 뒤 시험 기록을 삭제합니다.
- 네트워크가 불안할 때를 대비해 수동 바코드 입력 흐름도 확인합니다.

## 설명 원칙

- 국내 공공데이터를 먼저 조회하고 글로벌 데이터는 국내 결과가 없을 때만 참고합니다.
- 분리배출은 근거 수준을 세 단계로 표시하며 실제 포장 표시와 지역 기준이 최종 기준입니다.
- 영양 참고량은 2025 KDRI 기반이며 개인 진단이나 식사 처방이 아닙니다.
- 캘린더 합계는 기록된 음식만 합산하므로 하루 전체 영양 상태를 판정하지 않습니다.
