# 국내 제품 실물 검증 절차

이 문서는 앱에 제품을 하드코딩하는 목록이 아니라, HACCP 공공데이터 DB에서 조회 가능한 국내 제품을 실제 포장과 대조하기 위한 시험표입니다. 자동 조회가 성공해도 실물 검증이 끝난 것은 아닙니다.

## 자동 조회

서버 실행 후 다음 명령으로 `verification/domestic-products.json`의 15개 바코드를 검사합니다.

```bash
npm run verify:domestic -- http://localhost:3000
```

검사 항목은 바코드 검증 숫자, 제품 조회 성공, 국내 공공데이터 등급, 예상 제품명입니다.

## 실물 포장 대조

각 제품을 직접 준비한 뒤 아래 항목을 확인하고, 확인이 끝난 제품만 JSON의 `physicalPackageChecked`를 `true`로 바꿉니다.

1. 카메라가 바코드를 한 번에 인식하는가
2. 앱의 제품명과 제조사가 포장 표기와 같은가
3. 영양성분 기준이 `100g`, `1회 제공량`, `총 내용량` 중 포장 표기와 같은가
4. 실제로 먹은 양을 입력했을 때 열량과 영양소가 같은 비율로 계산되는가
5. 포장 재질이 앱에 표시되면 실제 분리배출 표시와 같은가
6. `공공 API 확인`, `재질 기반 추정`, `포장 표시 확인 필요` 등급이 과장 없이 표시되는가

제품이 단종되었거나 포장 표시가 공공데이터와 다르면 실패로 기록하고 다른 시중 제품으로 교체합니다. 실물 사진이나 표기 내용을 저장소에 올릴 때는 개인정보와 구매정보가 보이지 않도록 자릅니다.
